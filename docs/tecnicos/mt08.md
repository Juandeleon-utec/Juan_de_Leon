# Práctica 08: Preparación de archivo para router CNC

## Preparación de archivo para mecanizado CNC

### Introducción

En este ejercicio trabajé en la preparación de un archivo para mecanizado en router CNC a partir de un modelo provisto en formato `.f3d`, utilizando el módulo de fabricación de Autodesk Fusion 360.

El objetivo principal fue generar correctamente las trayectorias de herramienta (toolpaths) necesarias para fabricar la pieza en una placa de madera multilaminada de 200x200x10 mm (rescalado), aplicando estrategias de mecanizado 2D como:

- cajeras (pocket),
- taladrado (drilling),
- contorneado (contour),

así como definir parámetros críticos como:

- herramienta de corte,
- profundidad de pasada,
- sistema de coordenadas,
- uso de tabs (conexiones),
- generación de código G.

Más allá del resultado final, el ejercicio permitió comprender la relación entre el diseño digital, la simulación y el comportamiento real de la máquina CNC.


### Investigación y referencias

Antes de comenzar el proceso repasé conceptos clave del módulo:

- Configuración de máquinas CNC en entornos CAM  
- Definición de sistemas de coordenadas (WCS)  
- Estrategias de mecanizado 2D  
- Importancia de la simulación previa  
- Uso de tabs para evitar desprendimiento de piezas  

También tomé como referencia tutoriales y la documentación del propio Fusion 360 para comprender el flujo de trabajo dentro del módulo de fabricación.


## Proceso

### Configuración en Fusion 360

Se importó el archivo `.f3d` proporcionado y se accedió al entorno de fabricación.  
Se realizó un reescalado del diseño para que se adaptara al material que tenía disponible. Se lo reescaló al 50% del tamaño original, quedando una figura de 100x100x5 mm. Para fresar, se pegaron 2 hojas de OSB de 3 mm con cemento instantáneo del tipo metacrilato, para lograr una placa de 120x120x6 mm.

Se configuraron los siguientes aspectos:

- **Máquina:** fresadora tipo router CNC (similar a X-Carve)  
- **Origen (WCS):** ubicado en la esquina superior del material, lo más cercano posible al cero máquina  
- **Material:** madera multilaminada 120x120x6 mm  

Esta etapa es crítica, ya que una mala definición del origen o de los ejes puede provocar errores graves en la ejecución del mecanizado.

### Estrategias de mecanizado

Se implementaron las siguientes operaciones:

- **2D Cajeras:** para generar cavidades internas  
- **Taladrado:** para perforaciones específicas  
- **Contorneado:** para definir el corte exterior de la pieza  

Se utilizó una herramienta con:

- profundidad de pasada: **1 mm**,  
- adecuada para trabajar madera sin sobrecargar la fresa.

También se incorporaron **tabs (conexiones)** en el contorno final para evitar que la pieza se desprenda durante el mecanizado.


### Simulación

Se realizaron simulaciones de todas las operaciones dentro de Fusion 360 para verificar:

- correcta remoción de material,  
- ausencia de colisiones,  
- coherencia entre el modelo y el resultado mecanizado.

Esta etapa permitió validar el proceso antes de generar el código G.


## Dificultades encontradas

Durante el desarrollo del ejercicio surgieron problemas importantes que afectaron directamente el uso de Fusion 360 como herramienta CAM.

### Errores en la exportación del G-code en formato GRBL

Al generar el código G desde Fusion 360, se detectaron problemas graves:

- fallas en la exportación y no generación del G-code  
- movimientos inesperados de la máquina,  
- llamadas a posiciones inválidas,  
- desplazamientos bruscos dentro del material.

El primer error a destacar fue que, siguiendo el instructivo facilitado, utilizar un método de enfriado generaba un error, pero el mismo en ningún momento fue claro, únicamente truncaba el G-code. Reportaba un archivo como este:  
[Arcivo exportado con Error en fusion ](../anexos/mt08/1001.nc.failed)

Como se puede apreciar, no existe ningún tipo de log real que indique qué está pasando. Mediante prueba y error pude notar que, si no seleccionaba el método de enfriamiento, el error desaparecía y se generaba el G-code.

**Error más importante:**  
Generación de código con errores internos y no detectables, ya que en principio el G-code parecía correcto y el software de mecanizado tampoco mostraba nada fuera de lo común... pero...

Esto provocó que la CNC realizara movimientos incorrectos, llegando incluso a introducir la herramienta varios centímetros dentro del material, lo que ocasionó la **rotura de una fresa**.

Este tipo de error evidencia una incompatibilidad entre el postprocesador utilizado y la máquina CNC disponible.  
Intenté en su momento consultar con agentes de IA para que analizaran el código, pero ninguno detectó el error.  
También, después de mucho investigar, llegué a que el G-code contenía la línea: **G28 G91 Z0**. Según pude ver, eso accede a una posición de memoria que apunta a algún dato que la CNC tiene guardado. Pero esto lo descubrí después de retomar el tema.  
[Arcivo exportado con error de acceso a memoria ](../anexos/mt08/1002.nc)



## Solución adoptada

Debido a la repetición de errores y a la imposibilidad de obtener un G-code confiable desde Fusion 360, se decidió cambiar de estrategia.

Se optó por utilizar Easel, una plataforma web orientada a CNC, que permite:

- diseñar geometrías simples,  
- definir herramientas y profundidades,  
- generar y ejecutar el mecanizado directamente desde el navegador,  
- utilizar varias pasadas, con canalizaciones y perforaciones en el acrílico,  
- mientras que las letras y los diseños son con grabado superficial, el año "2026" se utilizó corte pasante para dejarlo perforado.

Se trabajó sobre el siguiente proyecto:

https://easel.com/projects/LhyMXo6ogfO5Z3miSnPEnA  

![Archivo generado](../images/Mt08/cnc.png)

Easel presenta ventajas importantes:

- integración directa con la máquina,  
- menor complejidad en la configuración,  
- entorno más robusto para trabajos de grabado.

Sin embargo, también tiene limitaciones:

- menor control sobre estrategias avanzadas de mecanizado,  
- menos flexible para fabricación de piezas complejas,  
- más orientado a grabado que a fabricación precisa,  
- trabajo realizado  


## Resultados

A partir del uso de Easel se logró:

- generar trayectorias estables,  
- evitar errores críticos de movimiento,  
- ejecutar mecanizados simples de forma segura.

Si bien no se replicó completamente el flujo de trabajo planteado en Fusion 360, se logró cumplir el objetivo práctico de comprender el proceso de preparación y ejecución en CNC.

## Reflexión personal

Este ejercicio dejó en evidencia que el flujo digital (CAD → CAM → CNC) no siempre es directo ni confiable, especialmente cuando intervienen:

- diferentes máquinas,  
- postprocesadores,  
- configuraciones específicas de hardware.

Más allá de seguir la consigna, el aprendizaje principal fue entender que:

> generar un G-code no es suficiente — debe ser compatible y seguro para la máquina real.

También resultó claro que herramientas más simples como Easel pueden ser más efectivas en ciertos contextos, especialmente en etapas de prototipado o cuando se prioriza la estabilidad sobre la complejidad.

El proceso, incluyendo errores y decisiones, fue fundamental para comprender las limitaciones reales del mecanizado CNC en entornos de bajo costo.
**Y que siempre se pueden descubrir errores nuevos. No hay nada muy claro que fusion 360 tenga algun tipo de incompatibilidad con el formato GRBL.**
